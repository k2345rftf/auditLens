# Вкладка «Обзор» → «Утренний брифинг аудитора» — полный план

> Статус: план утверждается. Подготовлен 2026-07-02 по итогам мульти-агентной проработки:
> инвентаризация данных, фактическая проверка источников с cloud.ru IP (94.139.252.160),
> 3 независимых дизайн-концепта, техническая архитектура.

---

## 1. Концепция

Сейчас «Обзор» — «позиция по цифрам» (Сбер vs медиана рынка), аудитору это мало что даёт.

Новый «Обзор» — **ежедневный выпуск, который ИИ уже прочитал за аудитора**. Аудитор
розничного бизнеса заходит утром и за 2–3 минуты видит ВСЁ: что случилось за
сутки/неделю в жалобах клиентов, тарифах конкурентов, регуляторике ЦБ, инцидентах и
схемах мошенничества — с формулировками «почему это важно аудитору» и кнопками
«разобраться».

Три принципа:
1. **ИИ курирует, а не юзер ищет.** LLM раз в день ранжирует сигналы и пишет
   человеческие формулировки. Числа при этом всегда детерминированные (SQL-детекторы),
   LLM только формулирует — не выдумывает.
2. **Один выпуск в день на всех.** Дайджест генерируется раз в сутки (утром или первым
   визитом), кэшируется в Postgres, все ~50 юзеров читают одно и то же. Экономия ~50×
   против «LLM на визит». Дата выпуска в шапке честно объясняет кэш.
3. **Честная деградация.** Секция упала → деградирует секция, не страница. LLM упал →
   сырые детерминированные сигналы. Источник за капчей → помечен как недоступный.

Мобайл: ③ в одну колонку (новости после карточек), ② — grid 2×3, ⑤ — `table.m-cards`.

### ① Masthead — передовица
- Eyebrow-строка в духе текущего «Issue N»: `БРИФИНГ №<день года> · <дата> · сводка 07:04 МСК`.
- **LLM-заголовок дня** в `t-display` (Instrument Serif), ключевая фраза `<em>` с `--accent`.
- Счётная строка выпуска: «2 риск-сигнала · 1 благоприятный · 6 новостей · тихих зон: 4».
- Кнопка ⟳ (ручной refresh, для админа).
- **Генерация (первый визит дня):** заголовок «печатается» — skeleton + `.dr-type-caret`,
  подпись «собираю сводку · ~1–2 мин», поллинг раз в 20 с. Блоки ②⑤⑥ рендерятся сразу.
- **LLM недоступен:** статичный «Сводка за 2 июля» + бейдж warn «ИИ-курирование
  недоступно — показаны детерминированные сигналы».

### ①b Молния (условная полоса)
Появляется ТОЛЬКО при сигнале `level:high` из `weekly_signals` или `urgent` новости.
Ping-точка (реюз `.rv-radar-live`) + одна строка + переход. Редкость = вес.
Нет полосы — тоже сигнал («спокойно»).

### ② Пульс дня — детерминированный тикер (0 токенов)
5–6 ячеек через hairline: ключевая ставка (+step-спарклайн 6 мес, из SOAP ЦБ),
Δ Сбер vs медиана (`/api/sber-vs-market`), жалобы 7 дн с дельтой (`weekly_signals.overall`),
изменения тарифов за 7 дн (`change_history`), спред «макс. вклад Сбера − КС»,
флаги качества. Каждая ячейка кликабельна → своя вкладка. Живёт при любом отказе LLM.
Закрывает предложение ТБ «динамика ключевой ставки vs ставки вкладов/кредитов».

### ③ Сводка дня — лента карточек-инсайтов (ядро)
4–6 карточек из дневного дайджеста, отранжированных LLM по аудиторской значимости.
Первая — lead на всю ширину, остальные grid 2 колонки.

Анатомия карточки:
- **Кикер**: 3px severity-полоса слева (`--neg` риск / `--warn` наблюдение / `--pos`
  благоприятно) + mono-uppercase тип (ЖАЛОБЫ/ТАРИФЫ/ГЕО/СТАВКИ/КАЧЕСТВО/…).
  Реализует «красное/зелёное» из предложений ТБ — сдержанно, цвет только в полосе.
- **Заголовок-инсайт** (Source Serif 4, 17–19px) — человеческая формулировка LLM.
- **so-what**: 1–2 предложения «почему аудитору важно» (регуляторный контекст, 115-ФЗ…).
- **Микро-глиф матрицы рисков**: SVG 18×18, решётка 3×3, точка = позиция
  «вероятность × влияние» (значения присваивает LLM при генерации). Tooltip
  расшифровывает. Полная матрица — визуальный шум для 4–6 сигналов (v2 — см. §9).
- **Mini-viz по kind**: всплеск → `Spark` 14 дн с пиком; тариф → delta-стрип
  `16.5% → 18.0% ▲+1.5`; гео → строка бара с городом и per-100k; ставка → dual-spark.
- **Provenance-строка** (обязательна): `banki.ru · 214 жалоб · медиана+2·MAD` —
  аудитор не доверяет выводу без метода.
- **Действия**: `[Разобраться →]` (переход с фильтром) и `[✦ Спросить ИИ]`
  (предзаполненный промпт, БЕЗ автоотправки).

Состояния:
- Сигналов < 4 → строка «тихих зон»: `✓ вклады · ✓ карты · ✓ география — без аномалий`.
- Совсем тихо → calm-карточка «За сутки резких сигналов не выявлено» (+2–3 нейтральных
  наблюдения — LLM всегда возвращает min 3 пункта).
- LLM упал → сырые сигнал-чипы (готовый паттерн `rv-radar-chips`): `Блокировка счёта ×2.4`.

Переходы (механика):
- `Разобраться →`: `sessionStorage.setItem("al-rv-prefilter", {bank,theme,city,days})`
  → `location.hash="reviews"`; в ReviewsPage читать ключ в инициализаторах useState.
- `✦ Спросить ИИ`: промпт собран на этапе генерации дайджеста (поле `ai_prompt`,
  0 токенов на клик) → `sessionStorage.setItem("al-ai-prefill", prompt)` → `#ai`;
  в AIPage на mount: `setQ(prefill)`, focus, НЕ отправлять автоматически.

### ④ Новости для аудитора (правая колонка, sticky)
6–10 позиций из дайджеста: тип-бейдж (`▍ЦБ` / `▍ИНЦИДЕНТ` / `▍СХЕМА` / `▍КОНКУРЕНТЫ` /
`▍СТАВКА`), однострочная LLM-выжимка + why_it_matters, `домен · время` mono,
клик → источник `target=_blank`. Сверху фильтр-чипы по типу.
Шапка честно показывает покрытие: `цб ✓ · банки.ру ✓ · tg ⚠ недоступен`.
Полный отказ → «За сутки новости не собраны» + кнопка «Повторить сбор»
(перегенерация ТОЛЬКО news-секции).

### ⑤ Тарифные движения недели (change_history)
Таблица (десктоп) / m-cards (мобайл): банк (BankAvatar, Сбер с accent-полосой),
продукт, `было → стало` mono с `.delta pos/neg`, дата. Топ-10 по |Δ|,
«Все изменения →» → `#market`.
**Детектор массового движения** (бэк): ≥3 банков сменили ставки одной категории за
48 ч → warn-бейдж «массовое движение — повод для анализа» + LLM почти наверняка
поднимет это карточкой в ③. Это ровно кейс ТБ «три банка ввели грейс-периоды».
Пустое: «Изменений за неделю не зафиксировано · под наблюдением 68 банков ·
последний сбор <дата>» — тишина как результат работы, не как отсутствие данных.

### ⑥ Подвал доверия
Mono-строка: свежесть каждого источника, флаги качества (клик → #quality),
стоимость дайджеста («3 LLM-вызова/день»), ссылка «Источники →».
Аудитор — профессиональный скептик: страница, декларирующая свои ограничения,
вызывает больше доверия.

## 4. Техническая архитектура

### 4.1 Хранение: Postgres, `migrations/010_daily_digest.sql`
Строка = дата × секция (секции независимы: генерятся/падают/перегенерируются отдельно):

```sql
CREATE TABLE IF NOT EXISTS daily_digest (
    digest_date   date        NOT NULL,           -- «день» в МСК
    section       text        NOT NULL,           -- headline|reviews_brief|news|reviews_pulse|tariff_moves|quality_ops|...
    payload       jsonb       NOT NULL,
    status        text        NOT NULL DEFAULT 'ok',  -- ok|degraded|stale|failed
    generated_at  timestamptz NOT NULL DEFAULT now(),
    llm_model     text, tokens_in int, tokens_out int, gen_ms int, error text,
    PRIMARY KEY (digest_date, section)
);
CREATE TABLE IF NOT EXISTS digest_run (
    digest_date date PRIMARY KEY,
    started_at timestamptz NOT NULL DEFAULT now(),
    finished_at timestamptz,
    status text NOT NULL DEFAULT 'running',       -- running|ok|partial|failed
    trigger text NOT NULL,                        -- morning|lazy|manual
    detail jsonb
);
```
История не чистится (365 × ~15 КБ — копейки) → архив «что было неделю назад» бесплатно.

Почему не файл и не кэш процесса: контейнер перезапускается (пересчёт = сжигание
токенов), нужен архив по датам и транзакционный claim от stampede.

### 4.2 Триггер: гибрид (утренний job + lazy catch-up), мьютекс = advisory lock
- **Утренний job** в lifespan (паттерн `alerts_background_loop`): спит до 07:00 МСК
  (`DIGEST_GEN_HOUR_MSK`), генерит. Плюс catch-up при старте контейнера, если уже позже.
- **Lazy**: `GET /api/overview/digest` при отсутствии сегодняшних строк запускает
  генерацию в фоне и СРАЗУ отвечает вчерашним дайджестом с `meta.refreshing=true`.
- **Stampede-защита двухуровневая**: процессный `asyncio.Lock` + сессионный
  `pg_try_advisory_lock(класс, day.toordinal())` — держим выделенный коннект всю
  генерацию: краш процесса = лок отпущен автоматически (нет вечного «in-progress»).
  `digest_run.status='running'` — витрина для UI, не мьютекс; старше 15 мин игнорируется.
- Юзер во время генерации видит вчерашний выпуск целиком + бейдж «Обновляется…»,
  поллинг раз в 20 с, плавная подмена. Никогда не пустой экран.

### 4.3 Пайплайн: 3 LLM-вызова/день, остальное SQL
Каждая секция — независимая корутина с `asyncio.wait_for` + try/except; результат
пишется в свою строку сразу. Хард-кап прогона 5 мин.

**SQL-секции (0 токенов, ~2–5 с):**
- `reviews_pulse` — реюз `reviews_dash.overview/trend/themes/weekly_signals("Сбербанк")`:
  KPI недели, всплески (медиана+2·MAD, окна 0-7/7-14/14-63 дн, сравнение с рынком,
  гео-концентрация), топ-темы с momentum. ВСЁ УЖЕ НАПИСАНО.
- `tariff_moves` — `change_history` за 7 дн: топ-дельты (`diff ? 'rate_pct'`, cast из
  строк), сводка по банкам, детект массового движения (≥3 банков, 1 категория, 48 ч),
  RATE_JUMP из quality_flag с пометкой «проверить руками», позиция Сбера
  (`v_sber_vs_market`). Обязательно `max(changed_at)` + `last_run` — иначе несвежий
  сбор выглядит как «всё спокойно».
- `quality_ops` — `quality_flag` за сутки (агрегация по code, не по строкам) +
  свежесть источников (`extraction_run`) + капчи из `captcha_pending`.

**LLM-секции:**
- `reviews_brief` (1 вызов, smart, ~7k in / 0.8k out) — сводка недели по жалобам:
  вход = weekly_signals + топ-темы + ~25 свежих жалоб + жалобы вне таксономии.
  Реюз промпта `reviews_llm.anomaly_brief` с рамкой «дневной дайджест».
- `news` (fetch → дедуп → 1 вызов, ~6k in / 1.2k out) — конвейер:
  (1) параллельный httpx-fetch, таймаут 10 с/источник: RSS ЦБ + banki.ru + frankmedia +
  t.me/s/-превью + SearXNG (bing+dogpile) по фикс-запросам («Сбербанк инцидент»,
  «хищение банк схема», «ЦБ предписание розница»);
  (2) дедуп БЕЗ LLM: нормализованный заголовок + хост;
  (3) окно 24–48 ч, кап ~40 позиций (`DIGEST_NEWS_MAX_ITEMS`);
  (4) один вызов «ты аудитор розницы Сбера: отбери релевантное, сгруппируй
  (регуляторика / инциденты и мошенничество / схемы / конкуренты), 1–2 предложения +
  why_it_matters + severity, JSON» → `_loose_json_loads`.
- `headline` (1 вызов, fast, ~2k in / 0.3k out) — передовица: заголовок дня + счётная
  строка + 4–6 карточек-инсайтов (ранжирование, so-what, prob/impact для глифа,
  ai_prompt) поверх УЖЕ записанных секций. Генерится последним.

**Бюджет: ~15k in / ~2.5k out токенов/день** (+ ручные refresh ×2–3).
**Время прогона: 1–2 мин** (fetch 15–30 с параллельно + SQL 2–5 с + 3 LLM-вызова).

### 4.4 Отказоустойчивость
- Источник новостей упал/капча → выпадает из корзины; `payload.sources[]` содержит
  `{name, ok, items, skipped_reason}` — фронт показывает покрытие. Секция всё равно ok.
- LLM недоступен → SQL-секции пишутся как обычно; `news` → status='degraded' с сырым
  списком заголовков; `reviews_brief`/`headline` → copy_forward вчерашней строки со
  status='stale' (+stale_from) — фронт вешает «сводка за вчера».
- Крашнулся прогон → advisory lock отпущен автоматически, следующий визит перезапускает.
- GET никогда не 500: отдаёт что есть, вплоть до `{sections:{}, meta:{empty:true}}`.

### 4.5 API
```
GET  /api/overview/digest              → сегодня (или последний доступный) + meta.refreshing
GET  /api/overview/digest?date=...     → архив (404 если нет)
GET  /api/overview/digest/dates        → список дат для архивного пикера
POST /api/overview/digest/refresh      → 202 {started} | 409 (уже идёт)
       body: {"force": true, "sections": ["news"]}    # точечная перегенерация
```
Фронт рендерит секции по ключам; неизвестный ключ → generic-карточка по
`payload.markdown`/`payload.items` — страница расширяема без релиза фронта.

### 4.6 Модуль `src/bank_audit/digest/`
```
digest/
├── store.py       # daily_digest/digest_run CRUD, copy_forward, try_acquire_day_lock
├── aggregator.py  # SQL-секции: reviews_pulse (делегат reviews_dash), tariff_moves, quality_ops
├── news.py        # NEWS_SOURCES=[{rss|tg|searxng}...], fetch_all() параллельно,
│                  #   normalize → dedupe → окно 48ч → топ-40; без LLM
├── writer.py      # LLM: reviews_brief, news_digest, headline; учёт usage.tokens
├── pipeline.py    # run_daily(day, force, only): SECTIONS-реестр, per-section
│                  #   wait_for+try/except+copy_forward; headline последним
└── scheduler.py   # digest_background_loop (07:00 МСК + catch-up), ensure_digest
```
Изменения: `web/app.py` (lifespan +1 task, +3 эндпоинта), `app.jsx` (OverviewPage:477
замена; +3-4 строки prefill в AIPage и ReviewsPage), `index.html` (CSS-слой `bf-*`),
`migrations/010_daily_digest.sql`.
Env-ручки: `DIGEST_GEN_HOUR_MSK=7`, `DIGEST_SECTION_TIMEOUT_S`, `DIGEST_NEWS_MAX_ITEMS`.

| Предложение | Решение |
|---|---|
| Алерты на всплески жалоб за сутки | ✅ Ядро блока ③ (`weekly_signals` уже реализован) + «Молния» |
| Новости-алерты ЦБ | ✅ Блок ④, RSS ЦБ проверен (P0) |

| «Три банка ввели грейс-периоды…» | ✅ Детектор массового движения в ⑤ + карточка в ③ |
| Интеграция журнала изменений тарифов | ✅ Блок ⑤ (change_history) |
| Ключевая ставка vs ставки вкладов/кредитов | ✅ Пульс ② + kind `rate_move` (SOAP ЦБ проверен) |
| Акции конкурентов | ✅ Тип «КОНКУРЕНТЫ» в ④, при значимости — карточка в ③ |
| Матрица рисков (вероятность × влияние) | ✅ Редуцированно: микро-глиф 3×3 на карточке. Полная матрица — v2, когда накопится архив дайджестов |
| Sunburst-дерево жалоб | ❌ Для «Обзора»: таксономия плоская, sunburst красив, но не даёт action; иерархию несёт риск-карта на «Отзывах». Кандидат на вкладку Отзывы |
| Прогноз трендов («ставка снизится на 10%») | ❌ LLM-прогноз без методологии в аудиторском инструменте подрывает доверие. Честная замена: флаг «ускоряется» (accel уже есть) |
| Макро-рейтинги bankiros.ru | ⏸ P2: HTML-парсинг есть, ценность для аудита средняя |
| Цифровые помощники с MAU | ⏸ Вне рамок «Обзора» |
| Быстрый поиск банка, полный текст жалоб, чистка нечитаемых | → относится к вкладке «Отзывы», отдельный мини-бэклог |

## 8. Этапы реализации

## 9. Риски и открытые вопросы

- **t.me/s/ может ограничить РКН** → деградация заложена (источник выпадает, покрытие
  видно в ④); не критично — RSS-ядро (ЦБ, banki.ru, frankmedia) независимо.
- **banki.ru за Qrator** → RSS отдался без капчи, но закладываем ту же деградацию.
- **Шум regex-таксономии** (отрицания «не навязывали») → рыночное сравнение
  компенсирует; формулировки пишет LLM по числам, не по отдельным текстам.
- **Несвежий сбор тарифов выглядит как «всё спокойно»** → обязательные `last_run` /
  `max(changed_at)` в ⑤ и подвале.
- **07:00 МСК и рестарт контейнера в 06:59** → catch-up при старте + lazy-триггер.
- Вопрос владельцу: нужен ли email-дайджест (у alerts уже есть SMTP) — та же
  секция headline письмом раз в день. Дёшево, но отложено до подтверждения.
